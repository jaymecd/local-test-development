404 - Route [HOSTNAME] not found

Few steps to fix the problem:

  1. verify docker compose stack is running,
    if not - start stack
  2. verify that stack services have traefik labels and proxy network attached,
    if not - adjust config and restart stack 
    # file: compose.yaml or docker-compose.yaml
networks:
  proxy:
    external: true
  my_stack:

services:
  app1:
    image: my_image:version
    labels:
      traefik.enable: true
      traefik.http.routers.my_app1.rule: Host(`[HOSTNAME]`)
      traefik.http.services.my_app1.loadbalancer.server.port: 8080  # container port
    networks:
      proxy:
      my_stack:
  ...
  3. didn't help? - recreate docker compose stack - down & up
  4. didn't help? - ask #experts

Good luck!
